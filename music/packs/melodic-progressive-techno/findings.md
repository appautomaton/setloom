# Findings: Reset for Rebuild

The previous findings file has been cleared as generator evidence.

It mixed public metadata, shallow genre assumptions, and early model review into
a lane-pack rulebook that became too restrictive. Treat every prior BPM range, key
distribution, duration window, reference-anchor summary, and set-construction
claim from this pack as deprecated until rebuilt from fresh evidence.

## Current Standing

| Area | Status |
| --- | --- |
| Metadata survey | Deprecated as creative authority |
| Named references | Vocabulary only, not targets |
| BPM/key/duration claims | Do not use as generator defaults |
| Set-flow claims | Rebuild from listening notes |
| Rejection rules | Technical hygiene only |

## Rebuild Shape

Future findings should be written only after a new reference pass exists. Each
claim must say whether it is:

- `observed`: heard or measured in a named reference at a timestamp.
- `hypothesis`: plausible but not yet validated by listening.
- `technical`: a production hygiene rule, not a musical style rule.
- `taste`: a human listening decision for a specific track.

No finding should become an executable generation default without a separate
design decision.
